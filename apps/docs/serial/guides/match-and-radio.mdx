---
title: "Match mode and radio"
description: "Read competition state and change controller radio channels."
---

```ts
import { RadioChannelType } from "@v5x/serial";

console.log(device.matchMode);
const updated = await device.setMatchMode("disabled");
if (updated.isErr()) throw updated.error;

if (device.radio.isAvailable) {
  console.log(device.radio.channel, device.radio.latency);
  const changed = await device.radio.changeChannel(RadioChannelType.PIT);
  if (changed.isErr()) throw changed.error;
}
```

`matchMode` is `"driver"`, `"autonomous"`, or `"disabled"`. Radio channels include `PIT` and `DOWNLOAD`.

`device.setMatchMode(mode)` returns `Ok(undefined)` on acknowledgement and
`Err(VexSerialError)` on NACK, timeout, or when the connection isn't open. The
`device.matchMode` setter is deprecated and will be removed in a future
breaking release.

<Warning>
  Do not change competition or radio state during an active match. Channel
  changes can interrupt communication.
</Warning>
