---
title: "Peripheral views"
description: "Read battery, controller, smart-device, button, settings, and radio state."
---

| View              | Important properties                                                          |
| ----------------- | ----------------------------------------------------------------------------- |
| `V5Battery`       | `batteryPercent`, `isCharging`                                                |
| `V5BrainButton`   | `isPressed`, `isDoublePressed`                                                |
| `V5BrainSettings` | `isScreenReversed`, `isWhiteTheme`, `usingLanguage`                           |
| `V5Controller`    | `batteryPercent`, `isMasterController`, `isAvailable`, `isCharging`           |
| `V5SmartDevice`   | `isAvailable`, `port`, `type`, `version`                                      |
| `V5Radio`         | `channel`, `isAvailable`, `isConnected`, `isVexNet`, `isRadioData`, `latency` |

These are state views created by `V5SerialDevice`. Get them from `device.brain`, `device.controllers`, `device.devices`, and `device.radio` rather than constructing them yourself.

`V5Radio.changeChannel()` changes between `RadioChannelType.PIT` and `RadioChannelType.DOWNLOAD`.

`V5Controller.isCharging` is `undefined` for the partner controller because the
V5 system-status response reports a charging bit only for the primary
controller.
