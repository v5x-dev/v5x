---
title: "CLI troubleshooting"
description: "Resolve project detection, toolchain, and serial connection failures."
---

<AccordionGroup>
  <Accordion title="Start with a local diagnostic report">
    Run `v5x doctor` to check Bun, platform support, installed toolchains, and
    visible serial ports before troubleshooting a specific project or hardware
    command. Use `v5x doctor --json` when sharing details for support.
  </Accordion>
  <Accordion title="v5x cannot detect the project">
    Run the command from the directory containing `project.pros`, the vexide
    `Cargo.toml`, or the VEXcode `vex` Make files. Alternatively, pass that
    directory as `path`.
  </Accordion>
  <Accordion title="The compiler executable is missing">
    Install the toolchain and check that Cargo, PROS, Python, or Make is on your
    `PATH`.
  </Accordion>
  <Accordion title="No V5 device is connected">
    Turn on the brain, reconnect its USB cable, close competing serial
    applications, and verify operating-system permissions.
  </Accordion>
  <Accordion title="The wrong brain is selected">
    Pass `--port <path-or-id>` to hardware commands such as `upload`, `run`,
    `devices`, `programs`, `dir`, and `screenshot`. Set `V5X_PORT` to reuse the
    same selector across commands.
  </Accordion>
  <Accordion title="An upload artifact cannot be found">
    Build the project first. When using `--no-build`, pass the intended `.bin`
    file with `--file`.
  </Accordion>
  <Accordion title="The screenshot is blank">
    Use a terminal implementing the Kitty graphics protocol. The command writes
    graphics escape sequences rather than an image file.
  </Accordion>
</AccordionGroup>
